# court-opinion — 庭审代理意见书

## 触发词
代理意见、庭审意见、出庭意见、代理词、庭审代理词、辩护意见、辩护词

## 功能说明
生成符合中国法庭提交格式的代理意见书（民商事/劳动）或辩护意见书（刑事），是出庭最核心的文书。

## 使用方式

### 步骤一：向 Claude 提供庭审信息
告知 Claude：
1. 案件类型（民事/刑事/劳动）
2. 代理方（原告/被告/上诉人/辩护人）
3. 案件事实陈述
4. 对方主张及证据
5. 己方证据情况
6. 核心法律争议点

### 步骤二：Claude 生成意见书 JSON
Claude 整理全部要素，输出结构化 JSON，保存为 `court_opinion.json`。

### 步骤三：生成 Word 文档
```bash
python3 ~/.claude/skills/court-opinion/scripts/generate_court_opinion.py \
  --input court_opinion.json \
  --output 代理意见书.docx
```

## Word 输出结构（符合中国法庭提交格式）
1. **标题**：《民事代理意见书》/《刑事辩护意见书》（黑体二号居中）
2. **代理人基本信息**（律师姓名、律师证号、律所、案号、庭审日期）
3. **代理意见正文**
   - 第一部分：事实意见
     - 己方事实陈述
     - 对对方主张的反驳
   - 第二部分：证据意见（对庭审中各方证据的综合意见）
   - 第三部分：法律适用分析（引用具体法条，论证法律适用）
4. **请求事项**（列明请求法院支持的具体诉求）
5. **落款**（代理律师签名 + 律师事务所 + 日期）

## 法条引用规则（必须遵守）

### 法条库优先
引用法条时，**优先**从 `~/.claude/skills/shared/references/法条库.json` 精确匹配，原文复制 `全文` 字段，并在条文末尾标注 `[库中已核实]`。

库中没有的条文，按以下三级确定性协议处理：

### 三级确定性标注协议

| 级别 | 含义 | 处理方式 |
|------|------|---------|
| `VERIFIED` | 可信度高、常用条文 | 直接引用，不加特殊标注 |
| `UNCERTAIN` | 不确定条文号或内容 | 在条文末尾加 `[待核实]`，禁止编造全文 |
| `OBSOLETE` | 已废止法律 | 禁止引用废止法律名称，必须替换为现行法 |

### 禁止行为
- **禁止**引用以下已废止法律名称（必须替换为括号内的现行法）：
  - ~~婚姻法~~ → 《民法典》第五编婚姻家庭
  - ~~合同法~~ → 《民法典》第三编合同
  - ~~侵权责任法~~ → 《民法典》第七编侵权责任
  - ~~物权法~~ → 《民法典》第二编物权
  - ~~继承法~~ → 《民法典》第六编继承
  - ~~担保法~~ → 《民法典》相关编章
- **禁止**生成超出法律实际条数的条文号（如《劳动合同法》正文止于第98条，《民法典》止于第1260条）
- **禁止**在不确定时编造条文全文

### JSON 输出格式要求
在输出 JSON 时，每条法条必须携带来源字段：
```json
{
  "法条": "《劳动合同法》第47条：经济补偿按劳动者...",
  "_来源": "LIBRARY",
  "_确定性": "VERIFIED"
}
```
或：
```json
{
  "法条": "《劳动合同法》第XX条（条文号待核实）",
  "_来源": "AI_GENERATED",
  "_确定性": "UNCERTAIN"
}
```

其中 `_来源` 取值：`LIBRARY`（来自法条库）或 `AI_GENERATED`（AI 生成，需核实）。

## 案例参考（可选）

分析案件时，可参考中国法院年度案例库中的真实判例：

1. 读取 `~/.claude/skills/shared/references/案例索引.json`
2. 按当前案件的领域/案由/关键词，检索相似案例
3. 如需案例详情，读取 `~/.claude/skills/shared/references/案例全文/{年份}/{册号}-{领域}/{序号}.txt`
4. 引用时标注：`[参考案例：{案例标题}，{裁判书字号}]`

注意：案例仅供参考，不构成判决依据。各地法院裁判尺度可能不同。
