# contract-review — 合同审查

## 触发词
合同审查、审合同、合同风险、条款审查、合同把关、审查合同

## 功能说明
Claude 逐条分析合同文本，提取风险点并生成结构化 Word 审查意见书，包含风险清单表、修改建议和综合意见。

## 使用方式

### 步骤一：提供合同信息
向 Claude 说明：
1. 合同文本（粘贴全文或上传文件）
2. 审查立场：甲方 / 乙方 / 中立
3. 合同类型：买卖/服务/劳动/房屋租赁/工程承包 等

### 步骤二：Claude 生成审查 JSON
Claude 分析后输出结构化 JSON，保存为 `review_result.json`。

### 步骤三：生成 Word 文档
```bash
python3 ~/.claude/skills/contract-review/scripts/generate_review.py \
  --input review_result.json \
  --output 合同审查意见书.docx
```

## Word 输出格式
1. 封面（合同名称、审查立场、审查日期）
2. 总体评价
3. 风险清单表（条款位置 | 风险等级 | 问题描述 | 修改建议）
4. 缺失条款说明
5. 综合建议

## 风险等级
- 高风险（红色）：可能导致重大损失或合同无效
- 中风险（橙色）：存在争议隐患，建议修改
- 低风险（黄色）：表述不规范，建议完善

## 法条引用规则（必须遵守）

### 法条库优先
引用法条时，**优先**从 `~/.claude/skills/shared/references/法条库.json` 精确匹配，原文复制 `全文` 字段，并在条文末尾标注 `[库中已核实]`。

库中没有的条文，按以下三级确定性协议处理：

### 三级确定性标注协议

| 级别 | 含义 | 处理方式 |
|------|------|---------|
| `VERIFIED` | 可信度高、常用条文 | 直接引用，不加特殊标注 |
| `UNCERTAIN` | 不确定条文号或内容 | 在条文末尾加 `[待核实]`，禁止编造全文 |
| `OBSOLETE` | 已废止法律 | 禁止引用废止法律名称，必须替换为现行法 |

### 禁止行为
- **禁止**引用以下已废止法律名称（必须替换为括号内的现行法）：
  - ~~婚姻法~~ → 《民法典》第五编婚姻家庭
  - ~~合同法~~ → 《民法典》第三编合同
  - ~~侵权责任法~~ → 《民法典》第七编侵权责任
  - ~~物权法~~ → 《民法典》第二编物权
  - ~~继承法~~ → 《民法典》第六编继承
  - ~~担保法~~ → 《民法典》相关编章
- **禁止**生成超出法律实际条数的条文号（如《劳动合同法》正文止于第98条，《民法典》止于第1260条）
- **禁止**在不确定时编造条文全文

### JSON 输出格式要求
在输出 JSON 时，每条法条必须携带来源字段：
```json
{
  "法条": "《劳动合同法》第47条：经济补偿按劳动者...",
  "_来源": "LIBRARY",
  "_确定性": "VERIFIED"
}
```
或：
```json
{
  "法条": "《劳动合同法》第XX条（条文号待核实）",
  "_来源": "AI_GENERATED",
  "_确定性": "UNCERTAIN"
}
```

其中 `_来源` 取值：`LIBRARY`（来自法条库）或 `AI_GENERATED`（AI 生成，需核实）。

## 案例参考（可选）

分析案件时，可参考中国法院年度案例库中的真实判例：

1. 读取 `~/.claude/skills/shared/references/案例索引.json`

4. 引用时标注：`[参考案例：{案例标题}，{裁判书字号}]`

注意：案例仅供参考，不构成判决依据。各地法院裁判尺度可能不同。
