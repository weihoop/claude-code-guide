# 图片转Word

将图片中的内容识别并转换为格式化的Word文档。

## 适用场景

- 硬件配置清单整理
- 表格数据转文档
- 截图内容转Word
- 报告/清单类图片整理
- 任何需要将图片内容结构化为文档的场景

## 工作流程

### 步骤1：读取并识别图片

使用 Read 工具读取图片文件，Claude 会自动识别图片中的内容结构。

```
Read 工具读取图片 → Claude 多模态识别 → 提取结构化数据
```

### 步骤2：分析内容结构

识别图片后，分析内容类型：
- **表格数据**：识别表头和数据行
- **配置清单**：识别配置项和对应值
- **文本内容**：识别标题、段落、列表
- **混合内容**：组合多种元素

### 步骤3：生成Word文档

使用 `scripts/create_word.py` 脚本生成带格式的Word文档。

**方式一：使用JSON数据生成（推荐）**

```bash
# 1. 将识别的数据整理为JSON格式
# 2. 运行转换脚本
python3 ~/.claude/skills/image-to-word/scripts/create_word.py <json文件路径> [输出docx路径]
```

**JSON数据格式示例：**

```json
{
  "title": "文档标题",
  "sections": [
    {
      "heading": "章节标题",
      "type": "table",
      "headers": ["列1", "列2", "列3"],
      "rows": [
        ["数据1", "数据2", "数据3"],
        ["数据4", "数据5", "数据6"]
      ],
      "header_color": "FFF2CC"
    },
    {
      "heading": "另一章节",
      "type": "text",
      "content": "段落内容..."
    },
    {
      "heading": "列表章节",
      "type": "list",
      "items": ["项目1", "项目2", "项目3"]
    }
  ]
}
```

**方式二：直接用Python生成**

对于复杂文档，可直接编写Python代码：

```python
from docx import Document
from docx.shared import Pt
from docx.enum.text import WD_ALIGN_PARAGRAPH

doc = Document()
doc.add_heading('标题', 0)
# ... 添加内容
doc.save('output.docx')
```

## Word文档格式规范

### 表格样式

| 元素 | 样式 |
|------|------|
| 表头行 | 加粗、居中、浅色背景 |
| 数据行 | 默认字体、居中对齐 |
| 边框 | Table Grid 样式 |

### 标题样式

| 级别 | 用途 |
|------|------|
| Heading 0 | 文档主标题 |
| Heading 1 | 一级章节标题 |
| Heading 2 | 二级章节标题 |

### 默认颜色方案

| 用途 | 颜色代码 | 说明 |
|------|----------|------|
| 蓝色表头 | D9E2F3 | 通用表格 |
| 黄色表头 | FFF2CC | 重点/核心数据 |
| 绿色表头 | E2EFDA | 成功/正常状态 |
| 灰色表头 | F2F2F2 | 次要信息 |

## 输出文件命名

- 默认：与内容相关的描述性名称
- 示例：`台式机配置清单.docx`、`账户信息汇总.docx`

## 常见文档类型模板

### 配置清单类

适用于：硬件配置、软件清单、资产列表

结构：
1. 文档标题
2. 基本信息表格
3. 核心配置表格（黄色表头突出）
4. 详细配置表格
5. 备注/评价

### 数据汇总类

适用于：财务数据、统计报表、数据导出

结构：
1. 报表标题
2. 汇总信息
3. 详细数据表格
4. 合计/小结

### 文档整理类

适用于：会议记录、操作手册、说明文档

结构：
1. 文档标题
2. 概述段落
3. 分章节内容
4. 列表/步骤
5. 附注

## 使用示例

用户提供图片后的处理流程：

1. 使用 Read 工具读取图片
2. 分析图片内容，确定文档结构
3. 整理数据为JSON格式或直接编写Python代码
4. 生成Word文档
5. 清理临时文件（虚拟环境、脚本等）
