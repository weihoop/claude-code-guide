# legal-doc-draft — 诉讼文书起草

## 触发词
起诉状、答辩状、上诉状、文书起草、仲裁申请书、诉状、劳动仲裁、强制执行

## 功能说明
根据案件信息生成符合中国法院/仲裁委要求的标准格式诉讼文书，支持 5 种文书类型。

## 文书类型

| 类型参数 | 文书名称 |
|---------|---------|
| `qisu` | 民事起诉状 |
| `daben` | 答辩状 |
| `susong` | 上诉状 |
| `zhongcai` | 劳动仲裁申请书 |
| `zhixing` | 强制执行申请书 |

## 使用方式

### 步骤一：填写对应模板 JSON
```bash
# 使用对应模板（5种文书各有模板）
cp ~/.claude/skills/legal-doc-draft/templates/起诉状模板.json ./案件信息.json
# 编辑案件信息.json，填入实际案件信息
```

### 步骤二：生成文书
```bash
python3 ~/.claude/skills/legal-doc-draft/scripts/generate_doc.py \
  --type qisu \
  --input 案件信息.json \
  --output 民事起诉状.docx
```

### 一次性命令（Claude 辅助填写）
向 Claude 说明案件基本情况，Claude 自动填写 JSON 并生成文书。

## Word 格式规范（符合中国法院要求）
- 标题：黑体二号，居中
- 当事人信息：表格形式，含姓名/身份证号/地址/联系方式
- 诉讼请求：编号列出（一、二、三……）
- 事实与理由：正文仿宋_GB2312，小四号
- 此致 + 法院/仲裁委名称
- 落款：代理律师/当事人 + 律师事务所 + 日期
- 页面：A4，页边距上下左右各 2.54cm

## 法条引用规则（必须遵守）

### 法条库优先
引用法条时，**优先**从 `~/.claude/skills/shared/references/法条库.json` 精确匹配，原文复制 `全文` 字段，并在条文末尾标注 `[库中已核实]`。

库中没有的条文，按以下三级确定性协议处理：

### 三级确定性标注协议

| 级别 | 含义 | 处理方式 |
|------|------|---------|
| `VERIFIED` | 可信度高、常用条文 | 直接引用，不加特殊标注 |
| `UNCERTAIN` | 不确定条文号或内容 | 在条文末尾加 `[待核实]`，禁止编造全文 |
| `OBSOLETE` | 已废止法律 | 禁止引用废止法律名称，必须替换为现行法 |

### 禁止行为
- **禁止**引用以下已废止法律名称（必须替换为括号内的现行法）：
  - ~~婚姻法~~ → 《民法典》第五编婚姻家庭
  - ~~合同法~~ → 《民法典》第三编合同
  - ~~侵权责任法~~ → 《民法典》第七编侵权责任
  - ~~物权法~~ → 《民法典》第二编物权
  - ~~继承法~~ → 《民法典》第六编继承
  - ~~担保法~~ → 《民法典》相关编章
- **禁止**生成超出法律实际条数的条文号（如《劳动合同法》正文止于第98条，《民法典》止于第1260条）
- **禁止**在不确定时编造条文全文

### JSON 输出格式要求
在输出 JSON 时，每条法条必须携带来源字段：
```json
{
  "法条": "《劳动合同法》第47条：经济补偿按劳动者...",
  "_来源": "LIBRARY",
  "_确定性": "VERIFIED"
}
```
或：
```json
{
  "法条": "《劳动合同法》第XX条（条文号待核实）",
  "_来源": "AI_GENERATED",
  "_确定性": "UNCERTAIN"
}
```

其中 `_来源` 取值：`LIBRARY`（来自法条库）或 `AI_GENERATED`（AI 生成，需核实）。

## 案例参考（可选）

分析案件时，可参考中国法院年度案例库中的真实判例：

1. 读取 `~/.claude/skills/shared/references/案例索引.json`
2. 按当前案件的领域/案由/关键词，检索相似案例
3. 如需案例详情，读取 `~/.claude/skills/shared/references/案例全文/{年份}/{册号}-{领域}/{序号}.txt`
4. 引用时标注：`[参考案例：{案例标题}，{裁判书字号}]`

注意：案例仅供参考，不构成判决依据。各地法院裁判尺度可能不同。
