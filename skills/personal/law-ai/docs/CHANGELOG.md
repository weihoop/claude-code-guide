## 2026-03-02 v1.1 — 案例参考库优化 + HTML可视化

### build_case_index.py Bug修复（12项）

**正则表达式修复（高优先级）**
- 修复 `被[执行]*申请人` 字符类错误 → `被(?:执行)?申请人`
- 修复 `申请[执行]*人` 同样问题 → `申请(?:执行)?人`
- 法条正则字符集加入"零"字，支持"第一千零八十七条"匹配
- 法条正则支持"第X条、第X条"多条连续引用格式

**容错性改进（中优先级）**
- `extract_pdf_text()` 单页提取失败不再中断整体，加 try-except 容错
- `find_case_boundaries()` 移除未使用的 `pages_with_nums` 参数
- `extract_volume_number()` 第二条正则收紧为 `r'^(\d{1,2})(?=\D)'`，加范围校验 1-23
- `clean_page_text()` 实现连续空行合并逻辑（此前注释说合并但未实现）
- `os.path.dirname()` 返回空字符串时加空值判断（2处）

**性能优化（低优先级）**
- 全文字符串拼接从 `full_text += text` O(n^2) 改为 list + join O(n)

**2024版OCR特殊处理（中优先级）**
- 新增 `_detect_reversed_text()` 函数：检测反向OCR文本（标点出现在行首）
- 新增 `_reverse_text()` 函数：自动翻转反向文本
- `find_case_boundaries()` 增加模糊标签匹配：严格匹配失败时退化为 `[【\[].{0,2}案件基本信息[】\]]`

### 新增 generate_case_report.py

- 读取案例索引JSON，生成单个自包含HTML文件（5.94 MB）
- 左侧领域导航栏（23个领域）
- 关键词搜索（标题/案由/裁判要旨/法条/当事人/法院）
- 年份筛选 + 领域筛选
- 点击展开案例详情（当事人、案情、焦点、裁判要旨、法官后语、涉及法条）
- 全文路径显示及一键复制
- 分批渲染优化（每批50条，避免DOM卡顿）
- 响应式布局 + 打印样式
- 配色参考法律文书风格（深蓝 #17376A + 白底）

### 帮助文档更新

- `generate_help_doc.py` 中"约800+判例"更新为"约1660+判例"（FAQ + 已完成项目表）
- 重新生成《法律AI技能使用手册.docx》
- `CLAUDE.md` 更新：共享资源表增加 generate_case_report.py、文件结构补充完整
