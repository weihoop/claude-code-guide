# case-summary — 案情梳理与法律意见书

## 触发词
案情梳理、法律意见、案件分析、出具法律意见书、法律意见书、案情分析

## 功能说明
根据案件事实和委托方信息，生成正式的法律意见书 Word 文档，包含案件事实概述、法律关系分析、争议焦点、法律意见、风险提示等完整内容。

## 使用方式

### 步骤一：告知 Claude 案件情况
向 Claude 描述：
1. 委托方基本信息（当事人身份、案件类型）
2. 案件事实经过
3. 委托方诉求或关注的法律问题
4. 现有证据情况

### 步骤二：Claude 生成意见书 JSON
Claude 分析后输出结构化 JSON，保存为 `opinion_data.json`。

### 步骤三：生成 Word 文档
```bash
python3 ~/.claude/skills/case-summary/scripts/generate_opinion.py \
  --input opinion_data.json \
  --output 法律意见书.docx
```

## Word 输出结构
1. **委托方基本信息**（当事人、案件类型、受托日期）
2. **案件事实概述**（时间线方式呈现）
3. **法律关系分析**（识别核心法律关系）
4. **争议焦点**（列举主要争议点）
5. **相关法律规定**（引用具体法条）
6. **法律意见**（分条论述，针对每个争议焦点）
7. **证据建议清单**（建议收集的证据目录）
8. **风险提示**（诉讼/仲裁风险分析）
9. **律师声明 + 落款**

## 法条引用规则（必须遵守）

### 法条库优先
引用法条时，**优先**从 `~/.claude/skills/shared/references/法条库.json` 精确匹配，原文复制 `全文` 字段，并在条文末尾标注 `[库中已核实]`。

库中没有的条文，按以下三级确定性协议处理：

### 三级确定性标注协议

| 级别 | 含义 | 处理方式 |
|------|------|---------|
| `VERIFIED` | 可信度高、常用条文 | 直接引用，不加特殊标注 |
| `UNCERTAIN` | 不确定条文号或内容 | 在条文末尾加 `[待核实]`，禁止编造全文 |
| `OBSOLETE` | 已废止法律 | 禁止引用废止法律名称，必须替换为现行法 |

### 禁止行为
- **禁止**引用以下已废止法律名称（必须替换为括号内的现行法）：
  - ~~婚姻法~~ → 《民法典》第五编婚姻家庭
  - ~~合同法~~ → 《民法典》第三编合同
  - ~~侵权责任法~~ → 《民法典》第七编侵权责任
  - ~~物权法~~ → 《民法典》第二编物权
  - ~~继承法~~ → 《民法典》第六编继承
  - ~~担保法~~ → 《民法典》相关编章
- **禁止**生成超出法律实际条数的条文号（如《劳动合同法》正文止于第98条，《民法典》止于第1260条）
- **禁止**在不确定时编造条文全文

### JSON 输出格式要求
在输出 JSON 时，每条法条必须携带来源字段：
```json
{
  "法条": "《劳动合同法》第47条：经济补偿按劳动者...",
  "_来源": "LIBRARY",
  "_确定性": "VERIFIED"
}
```
或：
```json
{
  "法条": "《劳动合同法》第XX条（条文号待核实）",
  "_来源": "AI_GENERATED",
  "_确定性": "UNCERTAIN"
}
```

其中 `_来源` 取值：`LIBRARY`（来自法条库）或 `AI_GENERATED`（AI 生成，需核实）。

## 案例参考（可选）

分析案件时，可参考中国法院年度案例库中的真实判例：

1. 读取 `~/.claude/skills/shared/references/案例索引.json`
2. 按当前案件的领域/案由/关键词，检索相似案例
3. 如需案例详情，读取 `~/.claude/skills/shared/references/案例全文/{年份}/{册号}-{领域}/{序号}.txt`
4. 引用时标注：`[参考案例：{案例标题}，{裁判书字号}]`

注意：案例仅供参考，不构成判决依据。各地法院裁判尺度可能不同。
